## 1. Context and goal

The M2.1 final whole-branch review found (I1) that four routes scoped out of M2.1 still
serve global credentials for any resolved ref, and the user decided to close them in a
fast-follow before M3 RBAC:

- `pages/api/platform/projects/[ref]/config/index.ts` and `config/postgrest.ts` — return
  the **global `AUTH_JWT_SECRET`** to the client for any ref (credential-bearing; a
  stronger credential than the `temporary.ts` leak M2.1 fixed — it mints unlimited
  service_role JWTs for the default stack).
- `pages/api/platform/projects/[ref]/api/rest.ts` and `api/graphql.ts` — proxy the
  default stack (`SUPABASE_URL` + `SUPABASE_SERVICE_KEY`, graphql also
  `SUPABASE_ANON_KEY`) for any ref (credential-using + wrong-stack data).

M2.2 also sweeps the final review's non-blocking follow-ups (user chose "全收") so M3
starts with a clean slate.

## 2. Scope

**In:** the 4 routes above; 5 hygiene code items (§4.2); 2 README notes (§5).
**Out:** `mcp.ts` projectRef threading (documented instead); review-package truncation
(plugin tooling, not this repo); everything M3 (RBAC, invites, per-user filtering).

## 3. Hard invariants (unchanged from M2.1)

Resolved registry row is the sole credential source (no env fallback on a hit); plain
self-hosted byte-identical with sibling `*.self-hosted.test.ts` guards; api-types
read-only, no `as any` (enum narrowing sole exception); zero new npm dependencies;
route errors top-level `{ message }`; `// [self-platform]` markers, minimal diffs.

## 4. Design

### 4.1 The four routes — conn.row-gated inline resolution (T6 pattern)

Chosen over a shared choke-point helper: only 2+2 routes with distinct response shapes
(YAGNI). Each handler: in self-platform mode call `resolveProjectConnection(String(req.query.ref))`;
on `conn.row` non-null use row values; on `conn.row` null (unregistered default) and in
plain self-hosted mode keep today's env-derived response byte-identically. Unknown ref →
`ProjectNotFound` propagates to the apiWrapper 404 net (`{ message: 'Project not found' }`).
All four export `handler` for tests.

- **`config/index.ts`** (GET): resolved → `jwt_secret: conn.jwtSecret`; all non-secret
  fields (`db_anon_role`, `db_extra_search_path`, `db_schema`, `max_rows`,
  `role_claim_key`) keep their current hardcoded values — the registry does not model
  PostgREST stack config, matching the `getProjectSettings(resolved)` precedent
  (`db_schema: 'public'` hardcoded). The weak-default fallback string remains only in
  the global branch (reachable only when `AUTH_JWT_SECRET` is unset). PATCH no-op
  unchanged.
- **`config/postgrest.ts`** (GET): same treatment; the `PGRST_DB_EXTRA_SEARCH_PATH` /
  `PGRST_DB_SCHEMAS` / `PGRST_DB_MAX_ROWS` env reads stay as-is in both branches
  (stack-level display config, not credentials, not registry-modeled). Response keeps
  `components['schemas']['GetPostgrestConfigResponse']`.
- **`api/rest.ts`** (GET; HEAD unchanged): resolved → `fetch(`${conn.supabaseUrl}/rest/v1/`)`
  with `apikey: conn.serviceKey`.
- **`api/graphql.ts`** (POST): resolved → `fetch(`${conn.supabaseUrl}/graphql/v1`)` with
  `apikey: conn.serviceKey` and `Authorization` = client's `x-graphql-authorization`
  header (existing array-coercion kept) falling back to `Bearer ${conn.anonKey}`.

### 4.2 Hygiene sweep (final-review follow-ups)

1. **Degradation once-guard** — `lib/api/self-platform/projects.ts`: module-level
   `let warnedMissingAnalyticsColumns = false`; log the operator hint once per process
   via `console.warn` (was `console.log` on every registry read on pre-M2.1 data dirs).
   Retry behavior unchanged.
2. **`temporary.ts` strict role type** — valid-JSON `claims` where `role` is present but
   not a string (e.g. `{"role":123}`) → 400 `{ message: 'Invalid claims parameter' }`
   (was: silently defaulted to `service_role`).
3. **Drains fallback error shape** — `analytics/log-drains.ts` + `log-drains/[uuid].ts`:
   in the self-platform branch, a missing-env fallback (unregistered default, no
   `LOGFLARE_*`) currently surfaces a bare `AssertionError` through apiWrapper's 500
   `{ error }`. Map `err.name === 'AssertionError'` in the existing in-route catch to
   500 `{ error: { message: err.message } }` — same shape as the plain branch's
   descriptive errors. Plain branch untouched.
4. **`analytics/endpoints/[name].ts` param validation** — array/missing `ref` or `name`
   → 400 `{ message: 'Invalid ref parameter' | 'Invalid name parameter' }` — **gated to
   IS_SELF_PLATFORM only**; the plain self-hosted branch keeps the original asserts
   byte-identically (zero-break is hard; the assert→400 change must not leak into plain
   mode).
5. **Pagination upper clamp** — `lib/api/self-platform/pagination.ts`:
   `parsePaginationParam` clamps results above 1000 down to 1000 (clamping philosophy per
   `temporary.ts`'s exp handling, rather than rejecting; only self-platform list routes
   consume this helper, so no zero-break surface).
6. _(moved to §5 — documentation items)_

### 4.3 Error handling summary

| Condition                                    | Behavior                                                                |
| -------------------------------------------- | ----------------------------------------------------------------------- |
| Unknown non-default ref (4 routes)           | ProjectNotFound → apiWrapper 404 `{ message: 'Project not found' }`     |
| Resolved row (4 routes)                      | Credentials/URLs from row only                                          |
| Unregistered default / plain mode (4 routes) | Today's env behavior byte-identical                                     |
| Non-string `role` in valid claims JSON       | 400 (self-platform; plain unchanged — plain branch never parses claims) |
| Drains missing-env fallback (self-platform)  | 500 `{ error: { message: '<assert message>' } }`                        |
| Array/missing ref/name on endpoints/[name]   | 400 in self-platform; original assert (500) in plain                    |
| `limit` > 1000                               | clamped to 1000                                                         |

## 5. Documentation (`docker/volumes/platform/README.md`)

1. **Shared-stack JWT secret note:** projects registered against the same stack share a
   JWT secret, so a temporary JWT minted for one ref verifies on its siblings — inherent
   to shared-stack registration; per-project secrets require separate stacks. Stated
   explicitly because M3 RBAC expectations get set here.
2. **mcp.ts asymmetry note:** in self-platform mode the MCP server's `getLogs` is
   per-ref (transitively via `retrieveAnalyticsData`) while its advisors/lints queries
   remain global-DB; projectRef threading deferred until MCP is multi-project. Update
   the boundary list accordingly (config/\* and api/rest|graphql move to "fixed in
   M2.2").

## 6. Testing

- The 4 routes have **zero tests today**: each gets handler tests (resolved-ref values,
  unknown-ref 404 via real apiWrapper + mocked apiAuthenticate, per the M2.1 sweep
  pattern) + a `*.self-hosted.test.ts` sibling (plain response byte-identical,
  resolver never called). vi.hoisted() TDZ precedent applies.
- Hygiene items: focused unit tests each — once-guard (console spy: warn fires once
  across two degraded calls), non-string role → 400, drains AssertionError mapping,
  [name].ts 400s (self-platform) + assert preserved (plain sibling), pagination clamp.
- Acceptance: full studio vitest + tsc 0 errors + lint 0 errors; live checks on the
  dual-project stack — `config/postgrest` for proj-b returns the registry-decrypted
  jwt_secret (equals default's in the shared-stack env — expected), ghost → 404,
  rest/graphql resolved refs hit the registry URL.

## 7. Decisions log

| Decision                           | Choice                                                                                  |

| Mechanism                          | conn.row-gated inline (T6 precedent); no new helper (YAGNI, 4 routes / distinct shapes) |
| Non-secret PostgREST config fields | Keep current values (registry doesn't model stack config; getProjectSettings precedent) |
| endpoints/[name] validation        | 400 gated to self-platform; plain keeps asserts (zero-break)                            |
| Pagination oversize                | Clamp to 1000 (not 400)                                                                 |
| mcp.ts                             | Document asymmetry only; no threading                                                   |
| Scope                              | Full hygiene sweep per user decision ("全收")                                           |
