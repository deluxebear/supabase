# M6.4 D3 — k8s container-metrics dialect Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Teach the Studio infra-metrics sampler to report a `stack_kind='k8s'` project's Postgres **Pod** CPU/RAM/network from a cAdvisor DaemonSet scrape, reusing the compose compute path.

**Architecture:** Generalize `computeContainerAttributes` from a raw `containerName` string to a `{cpuMem, net, cpuDenom}` `LabelPred` selector. A `stack_kind`-keyed `containerSelector(conn)` builds the compose selector (`name=` match) or the k8s selector (`io_kubernetes_pod_namespace`/`pod_name`/`container_name` match; CPU/mem on `container="postgres"`, network on the pod sandbox `container=""`). k8s identity lives in two new editable registry columns + the reused `container_name`. No route/frontend/`ATTRIBUTE_META` change.

**Tech Stack:** TypeScript, Next.js pages-router, Vitest, node-postgres via pg-meta, Prometheus text scrape.

## Global Constraints

- Spec: `docs/self-hosted-parity/2026-07-08-M6.4-D3-k8s-metrics-dialect-design.md`. Base: custom/main @814349b3ce.
- **Fixture-is-binding:** the k8s dialect binds `apps/studio/lib/api/self-platform/__fixtures__/cadvisor-k8s-scrape.prom`; never assert label/metric shapes not present in it.
- **Compose path stays byte-identical** in output — the selector refactor must not change any compose container-metric value (Task 3 has a guard test).
- k8s identity is **non-secret** — plain columns, no encryption/masking.
- Degradation ladder: column list ordered oldest→newest; the newest columns append **last**; the newest missing-column check runs **first**; a pre-migration db degrades new columns → `null`, never 500.
- Pod match is **exact** `pod_name` equality (no prefix/label/multi-replica selection).
- Run studio unit tests with `pnpm --filter studio exec vitest run <path>` (never `--filter studio test` — its coverage flag hangs).
- Tailwind + semantic tokens, `$t()` for user-facing strings, U.S. English.

- [ ] **Step 3: Add `k8sSelector` + `containerSelector`**

In `metrics.ts`, after `composeSelector` (Task 3), add:

```ts
// k8s dialect (cAdvisor on containerd). Identity is the io.kubernetes labels;
// cAdvisor's `name=` is a containerd task hex (useless). CPU/mem live on the
// workload container; NETWORK lives on the pod sandbox (container_name=""), and
// a second sandbox series per pod must be excluded from cpu/mem. Binds
// __fixtures__/cadvisor-k8s-scrape.prom.
export function k8sSelector(
  namespace: string,
  podSelector: string,
  container: string
): ContainerSelector {
  const pod: LabelPred = (l) =>
    l.container_label_io_kubernetes_pod_namespace === namespace &&
    l.container_label_io_kubernetes_pod_name === podSelector
  const workload: LabelPred = (l) =>
    pod(l) && l.container_label_io_kubernetes_container_name === container
  return {
    cpuMem: workload,
    cpuDenom: workload,
    net: (l) =>
      pod(l) && l.container_label_io_kubernetes_container_name === '' && l.interface !== 'lo',
  }
}

// Dialect choice for a resolved row: k8s identity → k8s; else container_name →
// compose; else null (host dialect, M6.3 behavior).
export function containerSelector(conn: {
  stackKind: string
  k8sNamespace: string | null
  k8sPodSelector: string | null
  containerName: string | null
}): ContainerSelector | null {
  if (
    conn.stackKind === 'k8s' &&
    conn.k8sNamespace != null &&
    conn.k8sNamespace !== '' &&
    conn.k8sPodSelector != null &&
    conn.k8sPodSelector !== '' &&
    conn.containerName != null &&
    conn.containerName !== ''
  ) {
    return k8sSelector(conn.k8sNamespace, conn.k8sPodSelector, conn.containerName)
  }
  if (conn.containerName != null && conn.containerName !== '') {
    return composeSelector(conn.containerName)
  }
  return null
}
```
