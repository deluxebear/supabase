## Global Constraints

- **Cloud byte-identical**: every SQL picker selects the EXISTING BigQuery text when built for cloud; substitution routes exist only in self-hosted deployments. Byte-identity is test-pinned.
- **Plain self-hosted intentionally improved** (spec D7, user-ratified): substitution + PG variants apply there too. v1 health keeps the M1 static stub byte-identically (probe changes self-platform-only).
- **Dialect gate constant** (SPEC REFINEMENT, record in ledger): the spec's `!IS_PLATFORM` shorthand is flag-algebra-corrected to `IS_SELF_PLATFORM || !IS_PLATFORM` — a self-platform build sets `NEXT_PUBLIC_IS_PLATFORM=true` AND `NEXT_PUBLIC_SELF_PLATFORM=true`; cloud is `IS_PLATFORM && !IS_SELF_PLATFORM`. Intent ("非 cloud 全修") unchanged.
- **Pinned Logflare BQ→PG translation rules** (live-bisected 2026-07-05/06, spike):
  - ✅ `countif`, `regexp_contains`, multi-level `cross join unnest`, field access on unnested records, `timestamp_trunc`, `group by <ordinal>` (incl. `group by 1, 2`), string-literal columns, `count(distinct …)`, `limit/order by <ordinal>`, `where` on CTE columns, JSON `metadata` field access.
  - ❌ `cast(… as datetime)` (`type "datetime" does not exist`), `group by <novel alias>`, **`UNION ALL` (categorically — translator only rewrites the first SELECT's table refs)**.
  - ⚠️ `group by <alias shadowing a column>` silently groups by the RAW column (fan-out, wrong data, no error) — **ordinals are mandatory**; a regression test pins this trap.
  - Timestamps return as **epoch microseconds** (16-digit); existing `isUnixMicro`/`unixMicroToIsoTimestamp` helpers normalize.
  - NOT YET PINNED (T2/T3 open with a live pin using the curl harness below): `json_value(event_message, '$.x')`, `count(distinct json_value(...))`, `safe_divide`, `cast(… as int64)`, `approx_quantiles(...)[offset(n)]`, `concat`/`case when … between`, `starts_with`, `where regexp_contains(...)`, `count(distinct case when …)`, `order by <ordinal> desc` after multi-ordinal group-by. Each pending idiom has a written fallback in its task.
